# Mini Recommendation Engine

## Overview

A simple recommendation system built from scratch to understand how recommendation algorithms work.

The project starts with traditional algorithmic approaches before gradually introducing machine learning concepts.

The goal is to understand:
- How user behavior becomes data
- How patterns are discovered from interactions
- How recommendation scores are generated
- How recommendation systems evolve into machine learning models


## Project Goals

### Phase 1 - Data Understanding
- Create and process user interaction data
- Represent user behavior using data structures
- Understand how raw data becomes usable information


### Phase 2 - Recommendation Algorithm
Implement a recommendation engine using:

- Frequency counting
- Co-occurrence analysis
- Ranking algorithms


Example:

If users frequently interact with:

Keyboard → Mouse

then the system learns:

Users interested in Keyboard may also like Mouse.


### Phase 3 - Improving Recommendations

Add:

- Weighted scoring
- Top-K recommendations
- User-based recommendations
- Item-based recommendations


### Phase 4 - Machine Learning Bridge

Extend the system toward ML concepts:

- Feature engineering
- Training data
- Prediction
- Model evaluation
